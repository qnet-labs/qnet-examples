StrategyType API Reference

Enum Definition​

from qnet-core import StrategyType

Members​

StrategyType.LowestLatency​

Minimizes end-to-end latency by preferring shorter paths and skipping expensive purification.

  • Best for: Time-critical protocols, high-fidelity links where purification is unnecessary
  • Behavior: Accepts raw link fidelity; does not trigger distillation cycles
  • Latency: Lowest among all strategies

StrategyType.HighestFidelity​

Maximizes output fidelity through purification (entanglement distillation) cycles.

  • Best for: QKD, quantum cryptography, blind quantum computing
  • Behavior: Triggers distillation when raw fidelity < target; consumes additional bandwidth
  • Latency: Highest among all strategies (purification overhead)

StrategyType.HighestSuccess​

Pragmatic strategy that maximizes the probability of success within constraints.

  • Best for: Noisy channels, capacity planning, reliability analysis
  • Behavior: Rejects requests early when no viable path exists; doesn't waste time on doomed attempts
  • Latency: Moderate (balances speed with rejection pragmatism)

Usage​

from qnet-core import Qnet-coreEngine, StrategyType

engine = Qnet-coreEngine()
# ... define network ...

result = engine.request_entanglement(
    from_node="Alice", to="Bob",
    fidelity_target=0.85, max_latency_ms=200.0,
    strategy=StrategyType.HighestFidelity,  # ← choose your strategy
)

Strategy Selection Flowchart​

Need maximum speed?          → LowestLatency
Need guaranteed quality?     → HighestFidelity
Channel is unreliable?       → HighestSuccess
Comparing configs?           → Use all three for full picture
