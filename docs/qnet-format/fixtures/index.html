.qnet File Format — Fixture Library

This repo ships with several .qnet fixture files for testing, examples, and learning. All fixtures use format version 1.0.

Available Fixtures​

network_v1.qnet — Simple 2-Node Fiber Link​

A minimal network: two nodes connected by a single fiber link.

{
  "version": "1.0",
  "metadata": { "name": "v1_network" },
  "nodes": [
    { "id": "node_a" },        // null optional fields → engine defaults
    { "id": "node_b" }
  ],
  "links": [{
    "from": "node_a",
    "to": "node_b",
    "distance_km": 100.0,
    "base_fidelity": 0.9,
    "generation_rate_hz": 100.0
  }],
  // config/constraints/extensions all null → use engine defaults
}
PropertyValue
Nodes2 (node_a, node_b)
Links1 (100 km fiber, fidelity 0.9, 100 Hz)
Use caseQuick tests, validation, baseline benchmark

network_v2.qnet — 3-Node Chain​

A chain topology: A → B → C with repeater at the middle node.

PropertyValue
Nodes3 (linear chain)
Links2 (fiber links)
Use caseRepeater protocol testing, multi-hop routing

valid_network.qnet — Validated Star Topology​

A valid star topology that passes all validation checks. Useful as a template for new networks.

PropertyValue
Nodes4 (1 hub + 3 edge nodes)
Links3 (star connections)
Use caseTemplate, validation reference

invalid_network.qnet — Intentionally Invalid​

A network with a self-loop (node_a → node_a). Used to test the validate() function.

{
  "links": [{
    "from": "node_a",
    "to": "node_a",   // ← self-loop! validate() will reject this
    "distance_km": 10.0,
    ...
  }]
}
PropertyValue
validate() resultvalid: false
ErrorsSelf-loop detected
Use caseTesting validation, error handling

toronto_london.qnet — Hybrid Satellite-Fiber Network​

A realistic intercontinental network simulating Toronto ↔ London via satellite.

{
  "version": "1.0",
  "metadata": {
    "name": "toronto_london_hybrid",
    "description": "Toronto to London hybrid network",
    "author": "user"
  },
  "nodes": [
    { "id": "Toronto", "memory_lifetime_ms": 150.0, "node_type": "ground" },
    { "id": "Montreal" },
    { "id": "London" },
    { "id": "SAT-1", "memory_lifetime_ms": 300.0, "node_type": "satellite" }
  ],
  "links": [
    { "from": "Toronto", "to": "SAT-1", "distance_km": 1000.0, "base_fidelity": 0.98, "generation_rate_hz": 50.0 },
    { "from": "SAT-1", "to": "London", "distance_km": 6000.0, "base_fidelity": 0.97, "generation_rate_hz": 50.0 }
  ],
  "config": {
    "alpha_loss": 0.04,
    "beta_fidelity_decay": 0.02,
    "gamma_swapping": 0.85,
    "max_attempts": 100
  },
  "constraints": {
    "fidelity_target": 0.85,
    "max_latency_ms": 50.0
  }
}
PropertyValue
Nodes4 (Toronto, Montreal, London, SAT-1)
Links2 (satellite uplink + downlink)
Total distance7,000 km
Simulation configalpha_loss=0.04, gamma_swapping=0.85
Global constraintsfidelity_target=0.85, max_latency_ms=50.0
Use caseSatellite communication example, intercontinental benchmark

Loading Fixtures from Examples​

from qnet-core import from_qnet_file, validate
import os

base = os.path.dirname(os.path.abspath(__file__))

# Load and simulate
engine = from_qnet_file(os.path.join(base, "../network_v1.qnet"))
stats = engine.simulate("node_a", "node_b", 0.7, 5000.0, 100)

# Validate a file
result = validate(os.path.join(base, "../invalid_network.qnet"))
print(f"Valid: {result['valid']}, Errors: {result['errors']}")
