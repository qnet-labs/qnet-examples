.qnet File Format — Schema (v1.0)

Top-Level Fields​

FieldTypeRequiredDescription
versionstringYesMust be "1.0"
metadataobject | nullNoHuman-readable metadata (all sub-fields nullable)
nodesarrayYesList of node definitions (min 1)
linksarrayYesList of link definitions (min 0)
configobject | nullNoGlobal simulation configuration
constraintsobject | nullNoGlobal constraints for all requests
extensionsnullNoReserved for future use (must be null)

metadata Object​

FieldTypeDefaultDescription
namestring | null—Network name
descriptionstring | null—Human-readable description
authorstring | null—Author / maintainer
created_atstring | null—ISO 8601 timestamp (e.g. "2026-06-12T12:00:00Z")

nodes Array​

Each node object:

FieldTypeDefaultDescription
idstringrequiredUnique node identifier

memory_capacityint | null—Memory slot capacity; null uses engine default

Each link object:

FieldTypeDefaultDescription

Validation Rules​

The validate() function checks:

  1. Version — must be "1.0"
  2. Nodes — at least one node; each node ID must be unique
  3. Links — all from/to references must exist as node IDs
  4. No self-loops — a link cannot connect a node to itself
  5. Required fields — distance_km, base_fidelity, generation_rate_hz on every link
  6. Fidelity range — base_fidelity must be in (0.0, 1.0]

Null Fields​

Null values are allowed for all optional fields. The engine falls back to sensible defaults:

FieldNull → Default
memory_lifetime_msEngine default (~150 ms)
memory_capacityEngine default (32 slots)
node_typeInferred from context
link_typeInferred from distance (>100 km → satellite, else fiber)
satelliteTreated as fiber physics model
